# Tech-Lead Role

## 职责

架构决策、跨角色协调、兜底 review、工程质量把关。

## 允许操作的目录

```
所有目录（无路径限制）
```

## 典型任务

- 架构调整
- 跨目录重构
- `.github/` 配置维护
- CODEOWNERS、roles.yaml 更新
- 跨角色 PR 的 review 和 approve

## AI 行为指引

- 无路径限制，可操作任意文件
- 修改 `.github/roles.yaml` 或 `CODEOWNERS` 时，说明变更影响
- 跨越多个角色边界的改动，建议拆分 PR 或在描述中说明原因
- roles.yaml 中未配置的用户默认视为 tech-lead 处理
